Explain and align the Offloadr product clearly across the website and app.

Important style rule:
Do not use emojis anywhere.
Do not use casual hype.
Do not use vague startup language.
Use clear, confident, professional wording.

Product name:
Offloadr

Core idea:
Offloadr is a post-recording workflow platform for podcast studios, schools, video creators, and production teams.

Offloadr does not record live audio or video.
Offloadr does not replace cameras, Rodecaster Pro II, Rode video switchers, OBS, SD cards, or local recording hardware.
Offloadr exists after the recording session is complete.

Main product sentence:
Offloadr turns raw recording sessions into clean, organised, ready-to-edit projects.

What problem Offloadr solves:
After a podcast or video recording session, files are usually scattered across multiple devices and sources. Audio may come from a Rodecaster Pro II, video may come from several cameras, the switched program record may come from a video switcher, and extra files may come from OBS, SD cards, USB drives, or editing software.

This creates a messy handoff for editors.

The editor often receives a pile of files with unclear names, missing camera angles, incomplete audio, no notes, and no obvious project structure.

Offloadr solves this by creating a structured workflow from raw files to editor-ready project handoff.

What the website/app should do:

1. Create a project
The user creates a new recording project or podcast episode.

The project setup should collect:
- Project name
- Episode title
- Recording date
- Client, school, or studio name
- Hosts
- Guests
- Expected number of cameras
- Expected audio setup
- Notes for the editor

This setup defines what files the project should contain.

2. Upload all local files
The user uploads all locally recorded files after the session.

Supported upload examples:
- Rodecaster Pro II audio exports
- Stereo mix
- Multitrack audio
- Individual microphone WAV files
- Program video recording
- Camera 1 file
- Camera 2 file
- Camera 3 file
- Camera 4 file
- OBS recording
- Screen recording
- Premiere Pro project file
- Final Cut XML
- DaVinci Resolve project file
- Exports
- Notes
- Thumbnails

The upload page should feel like the most important part of the product. It should be clean, large, obvious, and confidence-building.

3. Organise files automatically
After upload, Offloadr should detect file types and help sort files into a clean structure.

Suggested folder structure:
01_AUDIO
- Rodecaster Multitrack
- Stereo Mix
- Individual Mics

02_VIDEO
- Program Record
- Camera ISO
- Cam 1
- Cam 2
- Cam 3
- Cam 4
- Screen Recordings

03_PROJECT_FILES
- Premiere
- Final Cut
- Resolve
- Other

04_EXPORTS
- Final
- Drafts
- Social Clips

05_NOTES

06_BACKUPS

The app should allow manual tagging because automatic detection will not always be perfect.

4. File tagging
Each uploaded file should be taggable.

File role options should include:
- Rodecaster Multitrack
- Stereo Mix
- Host Mic
- Guest Mic
- Program Video
- Camera 1
- Camera 2
- Camera 3
- Camera 4
- Camera ISO
- Screen Recording
- Project File
- Final Export
- Thumbnail
- Backup
- Other

The point is not just to store files. The point is to make every file understandable to an editor.

5. Missing file checklist
Offloadr should compare the uploaded files against the expected setup.

Example:
If the project is a 4-camera podcast, the app should check for:
- Stereo mix
- Rodecaster multitrack
- Program video
- Camera 1
- Camera 2
- Camera 3
- Camera 4
- Editor notes

If something is missing, show a warning.

Allow the user to mark the project ready anyway, but make missing files obvious.

6. Ready for editor moment
This is one of the most important parts of the product.

When everything is uploaded and organised, show a strong completion state:

Project ready for editor

This page should show:
- uploaded file count
- total project size
- folder structure
- missing file warnings if any
- editor notes
- button to create editor link
- button to copy editor link after it is created

This should feel like the payoff of the entire workflow.

7. Editor handoff page
Offloadr should create a private editor handoff link.

The editor page should show:
- Offloadr logo
- Project name
- Episode title
- Recording date
- Client, school, or studio name
- Producer/editor notes
- Organised file tree
- File sizes
- Download links
- Project status
- Missing file warning if relevant

The editor should not need to ask:
- What is this file?
- Where is the audio?
- Which camera is which?
- Is anything missing?
- Where are the notes?

The handoff page should answer those questions clearly.

What Offloadr is not:
- It is not a live cloud recorder
- It is not Riverside
- It is not OBS
- It is not a video editor
- It is not a generic file uploader
- It is not just Dropbox with a new skin

What Offloadr is:
- It is a workflow system
- It is a project organisation tool
- It is a professional media handoff platform
- It is the bridge between recording and editing

Key positioning:
Dropbox and Google Drive store files.
Offloadr prepares recording sessions for editing.

The product should always be workflow-first, not file-first.
